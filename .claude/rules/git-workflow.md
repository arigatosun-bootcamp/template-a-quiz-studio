# Git ワークフロールール

コード変更時は必ず以下を確認すること：

1. **現在のブランチがmainでないこと**
   - mainなら先に `git checkout -b feat/issue-{番号}-{説明}` でブランチを切る
   - ブランチ名にはIssue番号を含める

2. **対応するIssue番号を把握していること**
   - Issue未作成なら先にIssueを作成する
   - `gh issue create` またはGitHub Web UIから作成

3. **コミットメッセージ規約**
   - `feat: 〜` — 新機能
   - `fix: 〜` — バグ修正
   - `docs: 〜` — ドキュメント
   - `test: 〜` — テスト
   - `chore: 〜` — その他（依存更新、設定変更等）

4. **コミットは小さく、頻繁に**
   - 1コミット = 1つの論理的変更
   - 「動く状態」でコミットする
