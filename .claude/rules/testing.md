# テストコード規約

## 必須要件

- テストは最低3本、必ず**異常系を1本以上**含める
- テスト名は日本語OK（何を検証しているか明確にする）
- `npm test` で全件パスすることを確認してからPR

## LLM出力テスト（Day3以降）

LLMを使う機能のテストには以下を必ず含める：
- 正常なJSON応答の処理
- 不正なJSON（パースエラー）の処理
- 空レスポンス / nullの処理
- タイムアウト / ネットワークエラーの処理
- バリデーション失敗時の再試行動作

## テスト構造

```typescript
describe('機能名', () => {
  // 正常系
  it('正常な入力で期待通りの結果を返す', () => { ... });

  // 異常系
  it('不正な入力でエラーを返す', () => { ... });

  // 境界値
  it('空文字列で適切に処理する', () => { ... });
});
```
