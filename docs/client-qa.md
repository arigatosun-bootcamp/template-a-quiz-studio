## AI・クイズ生成について

---

### Q1. AIはどこから問題を作っているの？

**回答例:**
「GoogleのGemini AIというサービスを使って、問題をリアルタイムに生成しています。
Geminiが学習済みの日本に関する知識（地理、観光地、グルメなど）をもとに、
毎回新しい問題文・選択肢・解説を作っています。」

**仕組みの詳細（自分の理解用）:**

```
ユーザーが「ゲームスタート」を押す
  ↓
アプリが Gemini AI に「こういう問題を作って」とお願い（これを"プロンプト"と言う）
  ↓
Gemini AI が問題文・3つの選択肢・正解・解説をJSON形式で返す
  ↓
アプリがそのデータをチェックして、問題なければ画面に表示
```

プロンプトの中身はこんな感じです:
```
あなたは日本に関するクイズの出題者です。
以下の条件で5問のクイズを作成してください。

【条件】
- ジャンル: 地理
- 難易度: 初級
- 出題する都道府県: 東京都、大阪府、北海道、沖縄県、京都府
```

つまり **問題のデータベースがあるわけではなく、AIがその場で考えて作っている** のがポイントです。
だから同じ条件で遊んでも、毎回違う問題が出ます。

---

### Q2. AIが間違った問題を作ることはないの？

**回答例:**
「AIなので100%正確とは言い切れませんが、いくつかの安全策を入れています。」

**具体的な安全策（自分の理解用）:**

1. **データ形式のチェック（バリデーション）**
   - AIの返答がちゃんとした形式か確認する
   - 問題文があるか、選択肢が3つあるか、正解が選択肢に含まれているか
   - 形式が不正ならエラーにして、ユーザーには「再試行」ボタンを出す

2. **リトライ（再試行）**
   - 生成に失敗したら自動で最大3回までやり直す
   - 1回目: 即座にリトライ
   - 2回目: 2秒待ってリトライ
   - 3回目: 4秒待ってリトライ
   - 3回全部失敗したら「もう一度お試しください」とエラー表示

3. **タイムアウト**
   - 10秒以内にAIが応答しなければ失敗扱いにしてリトライ

ただし **「問題の内容自体が正しいかどうか」のチェックはしていません。**
例えば「北海道の県庁所在地は？」で正解が「札幌市」なのは正しいですが、
AIが万が一「旭川市」を正解にしてしまう可能性はゼロではありません。
これはAIクイズの性質として説明するのが正直です。

### Q3. どうやって地域ごとの問題を分けて出しているの？

**回答例:**
「ユーザーの進捗を見て、まだ正解していない都道府県を優先的に5つ選び、
その県名をAIに渡して"この県の問題を作って"と指示しています。」

**仕組みの詳細（自分の理解用）:**

```
1. ユーザーが「地理 × 中級」でゲームスタート

2. アプリがデータベースから「このユーザーが中級以上でクリア済みの県」を取得
   例: 東京都、大阪府、北海道がクリア済み

3. 47都道府県 − クリア済み = 未クリアの県リスト
   例: 44県が未クリア

4. 未クリアの県からランダムに5県を選ぶ
   例: 沖縄県、福岡県、青森県、長野県、京都府

5. AIへのプロンプトに「出題する都道府県: 沖縄県、福岡県、青森県、長野県、京都府」と指定

6. AIがそれぞれの県に関する問題を1問ずつ作る
```

**ポイント:**
- 1回のクイズで同じ県は出ない（必ず5県別々）
- 未クリアの県が5つ未満の場合は、残りをクリア済みの県から補充する
- 全47県クリアしている場合は、全県からランダムに5県を選ぶ

### Q4. 問題の難易度はどうやって変えているの？

**回答例:**
「AIへの指示文（プロンプト）の中で、各難易度の基準を具体的に説明しています。
AI側がその基準に沿って問題の難しさを調整します。」

**プロンプトに書かれている基準:**

| 難易度 | AIへの指示内容 |
|--------|---------------|
| 初級 | 「誰でも知っている基本知識。有名な都道府県が中心。選択肢を見ればわかるレベル」 |
| 中級 | 「旅行好きなら知っている知識。やや細かい内容。少し考える必要があるレベル」 |
| 上級 | 「現地の人や詳しい人向け。マニアックな知識。知識がないと答えられないレベル」 |

難易度はAI任せなので **厳密にコントロールしているわけではありません。**
ただ、プロンプトで具体的に説明することで「だいたいそのレベル」にはなるようにしています。

## 利用制限について

---

### Q5. 1日最大何問まで解けるの？

**回答例:**
「1日最大100問です。1回5問 × 20回 = 100問が上限です。」

**仕組みの詳細（自分の理解用）:**

| 項目 | 値 |
|------|-----|
| 1回あたりの問題数 | 5問（固定） |
| 1日あたりの回数上限 | 20回 |
| 1日あたりの最大問題数 | 5 × 20 = **100問** |
| リセット時間 | 毎日 **日本時間 0:00** |

**上限チェックの仕組み:**
```
ゲームスタートを押す
  ↓
データベースで「今日このユーザーが何回セッションを遊んだか」を数える
（"今日" = 日本時間の0:00〜23:59）
  ↓
20回未満 → クイズ生成に進む
20回以上 → 「本日の利用上限に達しました。明日またお楽しみください」を表示
```

**なぜ20回制限なのか:**
- AI（Gemini API）の呼び出しにはコストがかかるため
- 無制限にすると大量のAPIコストが発生するリスクがある
- 1日100問あれば十分な学習量

---

### Q6. 制限時間はあるの？

**回答例:**
「ありません。1問ごとにじっくり考えて回答できます。」

制限時間なしにしている理由:
- 学習目的のアプリなので、焦らず考えてほしい
- 解説を読む時間も大切にしたい

---

### Q7. 「47都道府県を制覇」とはどういう意味？

**回答例:**
「各都道府県の問題に正解すると、その県が"クリア"になります。
47都道府県すべてクリアすることを"制覇"と呼んでいます。
さらに、初級・中級・上級の3段階あるので、最大で47 × 3 = 141パターンの制覇を目指せます。」

**制覇ルール（自分の理解用）:**

| ケース | 結果 |
|--------|------|
| 東京都の初級問題に正解 | 東京都が「初級クリア」になる |
| その後、東京都の中級問題に正解 | 東京都が「中級クリア」に更新される |
| その後、東京都の初級問題に正解 | 変化なし（中級のまま。下がることはない） |

データベースには「このユーザーの東京都の最高クリア難易度は中級」という形で保存されています。

### Q8. 日本地図の色は何を表しているの？

**回答例:**
「その都道府県をクリアした最高難易度を色で表しています。」

| 色 | 意味 | カラーコード |
|----|------|------------|
| グレー | 未クリア | #e5e5e5 |

## データ・セキュリティについて

---

### Q9. ユーザーのデータはどこに保存されているの？

**回答例:**
「Supabaseというクラウドサービスのデータベース（PostgreSQL）に保存しています。」

**保存されているデータ（5つのテーブル）:**

| テーブル名 | 保存内容 | 例 |
|-----------|---------|-----|
| users | ユーザー情報 | メールアドレス、登録日 |
| quiz_sessions | クイズのセッション | ジャンル、難易度、スコア、プレイ日時 |
| quizzes | AI生成された各問題 | 問題文、選択肢3つ、正解、解説、都道府県 |
| answers | ユーザーの回答 | 選んだ選択肢、正解かどうか、回答日時 |
| prefecture_progress | 都道府県の制覇状況 | 都道府県名、最高クリア難易度 |

---

### Q10. 他のユーザーのデータが見えることはない？

**回答例:**
「ありません。RLS（Row Level Security）という仕組みで、
自分のデータしか見えないようにデータベースレベルで制御しています。」

**RLSの仕組み（自分の理解用）:**
```
例えば answers テーブルには以下のルールが設定されている:

"Users can view own answers"
→ auth.uid() = user_id の場合のみ SELECT を許可

つまり、ログインしているユーザーのIDと一致するデータしか取得できない。
APIを直接叩いても、他人のデータは絶対に返ってこない。
```

---

### Q11. アカウントを消すとどうなるの？

**回答例:**
「アカウントと、そのユーザーに紐づくすべてのデータが完全に削除されます。復元はできません。」

**CASCADE削除の仕組み（自分の理解用）:**
```
ユーザーを削除すると、以下が自動的に連鎖削除（CASCADE）される:

users を削除
  ↓ CASCADE
  quiz_sessions が削除される（user_id で紐づいている）
    ↓ CASCADE
    quizzes が削除される（session_id で紐づいている）
      ↓ CASCADE
      answers が削除される（quiz_id で紐づいている）
  ↓ CASCADE
  answers が削除される（user_id で紐づいている）
  ↓ CASCADE
  prefecture_progress が削除される（user_id で紐づいている）
```

つまり、ユーザーを1件消すだけで、関連する何百件ものデータが一括で消える仕組みです。
消し忘れでデータが残ることはありません。

## 技術的な質問

---

### Q12. アプリが遅くなることはない？

**回答例:**
「クイズ生成時にAIの応答を待つため、数秒のローディングがあります。
それ以外の画面遷移やデータ表示は速いです。」

**具体的な待ち時間:**

| 処理 | 待ち時間 | ユーザーに見えるもの |
|------|---------|-------------------|
| クイズ生成（Gemini AI呼び出し） | 2〜5秒程度 | ローディングスピナー「クイズを生成中です...」 |
| 回答送信 | ほぼ一瞬 | ボタンが一瞬ローディング状態になる |
| 履歴・達成状況の読み込み | 1秒未満 | スケルトン表示（灰色の枠が出る） |
| ページ遷移 | ほぼ一瞬 | ページが切り替わる |

**10秒以上待っても応答がない場合:**
→ タイムアウトして自動リトライ（最大3回）
→ 3回失敗 →「問題の生成に失敗しました」とエラー表示 + 再試行ボタン

### Q13. スマホとPCで見た目は違うの？

**回答例:**
「基本的な機能は同じですが、レイアウトが画面サイズに合わせて変わります。」

| 項目 | PC（768px以上） | スマホ（768px未満） |
|------|----------------|-------------------|
| ヘッダーメニュー | 横並びで常時表示 | ハンバーガーメニュー（≡） |
| 日本地図 | 大きく表示 | 画面幅に合わせて縮小 |
| クイズ画面 | ゆとりのあるレイアウト | 縦に詰まったレイアウト |
| 操作 | マウスクリック | タップ |

## 答えに困ったときの対処法

---

### 「ちょっと確認して折り返します」で OK なケース

以下のような質問は、その場で答えられなくても問題ありません。

- 具体的な月額コストの見積もり
- 特定のセキュリティ基準への適合（ISMS、PマークなどI）
- 具体的なサーバーのスペックやリージョン
- 法的な問い合わせ（個人情報保護法への対応など）

**「確認して正確な情報をお伝えします」と言えば大丈夫です。**
