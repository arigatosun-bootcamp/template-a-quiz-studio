# L2 品質チェックリスト（LLM原則禁止）

コードを書く前に「これだけは絶対に守る」ルールを決めておく。
バグを出してから直すより、先に「壊れたらどうなるか」を考えておく方がずっと楽。

| 守るべきルール（例：空欄で送信できない） | なぜ必要？ | どうやって気づく？（例：テスト、目視） | 破れたらどうする？（例：エラー表示、ログ出力） |
|---|---|---|---|
空欄で送信できない｜押しミス等防止
漢字カタカナひらがな入力では送信できない｜押しミス等防止
絵文字、数字だけでは送信できない｜押しミス等防止
正しい内容が打たれなければログインできない｜不正ログインなどをなくす為
正式な内容を打ち込みログインする際すぐに表示できない時はリロード中とわかるようにする｜ユーザーの不反を減少させる　進んでいると伝える
打ち間違いがあった場合などはIDパスワードが違う旨を文面などで分かりやすく伝える　｜なんでログインできなかったのかを伝える
PWを忘れてしまった際の案内をテキストで表示す｜パスワードは忘れてしまうことがあるので忘れた際の保険
| 未ログインユーザーがデータを見られないようにする |自分の回答のみ確認できるようにする為  |ログインが必要なことをテキストなどの表示で伝える  |エラー表示を出す  |
難易度を選んだ際に選んだものがわかるようにする　ちゃんと選べていることを伝えるため
難易度を選んだらスムーズにクイズがスタートする　スタートまでの時間が長いとゲームに飽きられてしまう
ゲームが始まるとスムーズに５問問題が考えられて表示される　問題がスムーズに出ないとゲームにならない
５問回答が終わると何問正解したか表示される　すぐにゲームが終わると呆気ない
| 回答した時に答えが表示される |ここが表示されないとクイズにならない |テスト  |エラー表示を出す  |
|



  |  |  |  |

### 書き方のヒント
- 「〜できない」「〜してはいけない」のような **NG条件** で考えると書きやすい
- 例：「未ログインユーザーがデータを見られてはいけない」→ 認証チェックが必要
