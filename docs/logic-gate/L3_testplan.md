# L3 テスト計画（LLM原則禁止）

「ちゃんと動く」を確認するためのテスト項目を書く。
3種類をバランスよく混ぜるのがコツ。

- **正常**：普通に使ったときに期待通り動くか
- **異常**：おかしな入力や操作をしたときにちゃんとエラーになるか
- **境界**：ギリギリの値（0件、上限ちょうど、空文字など）で壊れないか

| ID | 種別 | 何をする？（操作や入力） | どうなれば正解？ | なぜこのテストが要る？ |
|---|---|---|---|---|
| T01 | 正常 |  |  |  |
| T02 | 異常 |  |  |  |
| T03 | 境界 |  |  |  |

### 書き方のヒント
- 他の人がこの表だけ見て、同じ操作を再現できるくらい具体的に書く
- 「正しく動く」ではなく「〇〇が表示される」「△△のエラーメッセージが出る」のように書く
