# L3 テスト計画（LLM原則禁止）

「ちゃんと動く」を確認するためのテスト項目を書く。
3種類をバランスよく混ぜるのがコツ。

- **正常**：普通に使ったときに期待通り動くか
- **異常**：おかしな入力や操作をしたときにちゃんとエラーになるか
- **境界**：ギリギリの値（0件、上限ちょうど、空文字など）で壊れないか

| ID | 種別 | 何をする？（操作や入力） | どうなれば正解？ | なぜこのテストが要る？ |
|---|---|---|---|---|
| T01 | 正常 | ログインIDとPWを入力する |ログインが完了しアプリTOPページに遷移 |ログインできないと回答の履歴を残せないから  |
| T02 | 異常 |  適当なログインIDとPWを入力する|エラーが表示されて再度ログインIDとPWを求められる  |適当に書いてログインできてしまうと本来残したい回答履歴などが表示できなくなる  |
このアプリのURLから遷移した際ログイン認証される
ログイン認証では正式な内容以外は送信できない仕様になっっている
PWを忘れた際のPW変更部分を押して進んでいくとPWが変更できる
正式な内容が打ち込まれ送信ボタンを押すとアプリのTOPに遷移できる
アプリのトップでは表示がきれいに表示される
難易度選択を押したら色が変わり押したことがわかるようになる　
難易度選択を押したらスタートテキストが表示されてクイズがスタートする
クイズが始まったら問題と回答用テキストボタン３種類が表示される
回答ボタンを押すと正解かっ不正解が表示される
その後次の問題に進む
５問完了後５問中何問正解できたか表示される
ゲーム終了後画面からTOPに戻るボタンが表示される
TOPに戻るボタンを押すことでTOPに戻れる
履歴ボタンで過去の問題と回答を見ることができる
履歴表示中にもTOPに戻るボタンが表示されている
履歴表示中のTOPに戻るボタンでTOPに戻ることができる

### 書き方のヒント
- 他の人がこの表だけ見て、同じ操作を再現できるくらい具体的に書くß
- 「正しく動く」ではなく「〇〇が表示される」「△△のエラーメッセージが出る」のように書く
