# L1 要件分解（LLM原則禁止）

## 1. 目的（誰の・何の困りごとを解決する？）
- 背景（なぜ作る？）：日本に住んでいるのにもかかわらず、日本の魅力を知ることなく生活している人に日本地図や都道府県、地域の観光名所など日本についての理解をただ覚えるのではなくクイズゲームで楽しく知識を深めるきっかけ作りたいと思い作成します
- 「できたらOK」と言える状態：クイズのサイトに遷移しユーザー認証ができてゲーム開始・終了と過去に解いた問題や自分の正解不正解の復習ができればOK

## 2. やること / やらないこと
### 今回やること　日本に関するクイズ　そのクイズはAIが自動的に５問問題を生成答えが3つ用意されてその中からユーザーが答えを回答する　問題は日本地図や日本の地名、県庁所在地や観光名所、ご当地のグルメについてなどが問題になり出題されます　問題には三段階のレベルから選ぶことができそのランクによって出題レベルが変わるようにしたいですTOPには日本地図が表示されていて問題を解いたことがある地域（正解したもの地域）に色がつくようにして日本制覇ができるようにしたいです　三段階のレベルによって地域につく色も変わるようにしたいです　問題は毎回新しいものだけでなくすでに出題済みの内容でも可能　回答が溜まれば溜まるほどその履歴で日本の教科書のような存在になる
-

### 今回やらないこと（次回以降 or 対象外）
-直接問題を指定する　日本についての検索機能　SNS共有機能　マイページの作成　

## 3. 使い方の流れ（ユーザー目線で）
ユーザーが何をして → アプリが何をして → どんな結果が返るか、を書く。

- 流れ1：ユーザーがアプリを開く　アプリからのユーザー認証をする　アプリが開きTOP画面で三段階の中からレベルを選んでゲームをスタートする　AIが問題を５問作成して順番にクイズが始まる　そのクイズに答えて正解か不正解かのアクションがあり　５問が終了したら５問中難問正解したか表示されてTOPに戻る
- 流れ2：ユーザーがアプリを開く　アプリからのユーザー認証をする　アプリが開きTOP画面で回答履歴を見る　過去の問題と回答が見れる

どんな情報をデータベースに保存する必要があるか、ざっくり洗い出す。

- 保存するもの（例：ユーザー情報、クイズの回答履歴など）：ユーザー情報　クイズの回答履歴　回答に基づいた日本地図の色付け情報
- なぜ保存が必要？（例：次回ログイン時に使う、集計に使うなど）：次回ログイン時に使う　過去のクイズの回答を確認できる　回答が溜まれば溜まるほどその回答が日本の教科書のような役割を果たす　TOPで表示する日本地図に色をつけるために情報の保存が必要　ここはどれだけ知識が増えたかなどを見た目からも理解できるようにするため
