# L2 品質チェックリスト（作成例）

コードを書く前に「これだけは絶対に守る」ルールを決めておく。
バグを出してから直すより、先に「壊れたらどうなるか」を考えておく方がずっと楽。

| 守るべきルール | なぜ必要？ | どうやって気づく？ | 破れたらどうする？ |
|---|---|---|---|
| メールアドレスが空欄のまま送信できない | 空っぽで登録されるとログインできなくなるから | テスト（空欄で送信ボタンを押して試す） | 「メールアドレスを入力してください」と赤文字で表示する |
| パスワードが8文字未満で登録できない | 短すぎるパスワードは簡単に破られてしまうから | テスト（7文字のパスワードで登録を試す） | 「パスワードは8文字以上で入力してください」と表示する |
| 間違ったパスワードではログインできない | 他人のアカウントに入れてしまうと個人情報が漏れるから | テスト（わざと間違ったパスワードを入力する） | 「メールアドレスまたはパスワードが正しくありません」と表示する |
| 未ログインの人がクイズ画面や履歴画面を開けない | 他人のデータが見えてしまう・誰の回答かわからなくなるから | テスト（ログインせずにURLを直接入力してアクセスする） | ログイン画面に自動で飛ばす（リダイレクト） |
| パスワードを忘れたときにリセットできる手段がある | パスワードは誰でも忘れることがあるから | 目視（ログイン画面に「パスワードを忘れた方はこちら」のリンクがあるか確認） | リセットメールが届き、新しいパスワードを設定できる |
| ジャンルと難易度を選ばずにゲームを開始できない | 何の問題を生成すればいいかAIが判断できないから | テスト（未選択の状態でスタートボタンを押す） | スタートボタンを押せない状態にする（グレーアウト） |
| クイズ生成中はローディング表示が出る | 何も表示されないとアプリが止まったと思ってユーザーが離脱するから | 目視（ゲームスタート後にぐるぐる回るアイコンが出るか確認） | 「問題を生成しています...」の文字とローディングアニメーションを表示する |
| AI（Gemini）がエラーになっても画面が真っ白にならない | ネットワークの問題やAPI制限でAIが応答しないことがあるから | テスト（APIキーをわざと無効にして生成を試す） | 「問題の生成に失敗しました。もう一度お試しください」と表示し、トップに戻るボタンを出す |
| 同じ問題に2回回答できない | 1問に何度も答えるとスコアがおかしくなるから | テスト（同じ問題のIDで2回回答APIを呼ぶ） | 「この問題はすでに回答済みです」とエラーを返す |
| 1日20回を超えてプレイできない | AIのAPI利用にお金がかかるためコストを管理する必要があるから | テスト（20回プレイ後にもう1回始めようとする） | 「本日の利用上限（20回）に達しました。明日またお楽しみください」と表示する |
| 回答履歴には自分のデータだけが表示される | 他人の回答が見えるとプライバシー違反になるから | テスト（2つのアカウントでプレイし、互いの履歴が見えないことを確認） | データベースのRLS（行レベルセキュリティ）で自分のデータのみ取得する |
| アカウント削除時に確認ダイアログが出る | 間違えて押した場合にデータが全部消えてしまうから | 目視（削除ボタンを押したときに「本当に削除しますか？」と出るか確認） | 確認ダイアログで「削除する」を押さない限り削除しない |
| スマホでも画面が崩れない | スマホから使うユーザーが多いから | 目視（ブラウザの幅を狭くして各画面を確認） | ハンバーガーメニューに切り替え、ボタンやテキストが画面内に収まるようにする |
